Rebase guidance

Rejected pull request from fork

To start, I have to say I LOVE GITHUB. You guys rock. I was at your 4th birthday party, thanks for the cupcake and booze!

Anyway, here's my problem. I sent a pull request to technoweenie, and he rejected it because my master branch is out of sync. What can I do to fix this?

Help me supportocat, you're my only hope!

/.Steve

Hi Steve,

We're excited to have you and your support! Nothing says 'celebrate' like chocolate and liquor! :-D

Correcting an out of sync branch is a two-step fix. First, get the latest code from master. This assumes master is pointed to technoweenie's branch

$ git pull origin master

Then,

$ git rebase -i origin/master

Rebasing is the term for adding your code to the most recent master. The -i means interactive so you can squash commits into one, which is what is typically preferred.

There's a great post titled about git rebase that covers a lot of this. Feel free to send me an issue and we can work through this together if you run into any more problems.

Adding Remotes

Syncing internal server and GitHub?

Hi GitHub! My company has an internal Git server used for deployments, we use GitHub, and I work on a local copy of the repository. For workflow purposes, how do I sync my repository to GitHub and the internal Git server?

Thanks,

vmg

Hey VMG,

How's your day going?

To connect your local repo to any remote server first get to the root directory and run the following

$ git remote add origin internal-server [repo url]

then for a GitHub remote

$ git remote add origin github [repo url]

note the [repo url] is usually the SSH address like

git@github.com:vmg/name-of-repo

Here's a short post titled Adding a remote which goes into more detail.

Fork Policy

Unhappy about forking

JamesTK here, I have another question.

One of my dev collaborator forked my private repo without explicit permission to do so... is that typical? I am surprised that the system did not notify me if it was okay to grant the fork permission. Is there a way to setup permissions of this sort? Also, other than asking the collaborator to delete the forked repo, can I actually delete it?

Thanks, jamestk

James,

Collaborators have the ability to fork repos they contribute to since

forking is a key component to team collaboration. If you remove the collaborator from your private repo, this will also remove the collaborator's fork However, that is really for situations where the collaborator no longer works on the project.

Regarding;

Is there a way to setup permissions of this sort?

Great question! Any new private repo you create begins with only yourself. 'Permissions' are a topic of team management. Here are two resources for help with that:

1. Intro to setting up teams.

2. Ongoing management and maintaining teams.

Lastly;

Also, other than asking the collaborator to delete the forked repo, can I actually delete it?

Unfortunately you are not able to delete another user's fork. You might find this page about forking helpful to get a better idea how the relationship works.

I'm here if you get stuck!
